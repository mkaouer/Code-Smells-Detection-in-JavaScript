Lines of Comments (LOCOMM)

This metric is defined as the number of multi-comment and single-comment lines. The number of lines of comments influences readability, self-descriptiveness and understandability of a piece of software. Under commenting has a great negative impact on code understandability. But over commenting adversely affects productivity by diverting effort away from the actual development of the code.

  Known thresholds
Used in conjunction with the LOC metric, a percentage (Average Lines of Comments per Method) can be obtained in order to assess the quality of a system. Code where the percentage of comment is lower than 60% should be more commented. However overly commented code (>80%) is not necessarily a blessing as some information might be redundant or obvious. In procedural code and object-oriented code with method names that are not self-descriptive, the minimum value for comment density should be 10 to ensure that software engineers can adequately determine the function of the code during maintenance. In object-oriented code with self-descriptive method names, the minimum value for comment density for functions, methods and procedures with complexities greater than 4 should be 10 to ensure that software engineers can adequately determine the function of the code.
