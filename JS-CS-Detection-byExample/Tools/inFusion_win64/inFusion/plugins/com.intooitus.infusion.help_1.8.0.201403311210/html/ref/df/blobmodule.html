Blob Module

The Blob Module design flaw is the equivalent of Blob Class in the procedural world. It designates a module that has a large size and a high complexity. These characteristics have a negative impact on the maintainability of the affected module.

Detection rule

Blob Modules are detected using a combination of size, complexity and cohesion metrics, as well as the detection rule for Blob Operation. The detection rule for Blob Module differs only slightly from that for Blob Class, in that unlike in the object-oriented world, in the procedural world a long parameter list for functions is not uncommon, thus it does not uses the NOPAR metric to look for functions with a higher than average number of parameters.
