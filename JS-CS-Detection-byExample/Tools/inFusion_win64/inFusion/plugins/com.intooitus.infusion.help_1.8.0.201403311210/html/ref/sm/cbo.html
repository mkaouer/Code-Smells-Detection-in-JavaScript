Coupling Between Objects (CBO)

This metric is defined as the number of distinct number of other classes from which methods are called, or attributes are accessed by the methods of the measured class. A high CBO value is undesirable because it indicates an excessive coupling between object classes, which is detrimental to modular design and prevents reuse. The more independent a class is, the easier it is to reuse it in another application. In order to improve modularity and promote encapsulation, the coupling between object classes should be kept to a minimum. The larger this coupling, the higher the sensitivity to changes in other parts of the design, which adversely affects maintenance. Furthermore, a high coupling has also been found to indicate proneness to faults, which in turn leads to higher testing effort.
