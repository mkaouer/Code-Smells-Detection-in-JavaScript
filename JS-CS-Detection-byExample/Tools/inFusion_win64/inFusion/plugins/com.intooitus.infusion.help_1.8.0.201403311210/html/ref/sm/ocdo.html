Outgoing Coupling Dispersion for an Operation (OCDO)

This metric is defined as the number of distinct (external) classes or modules from which operations (methods or functions) are called by the measured operation. Operation calls are a form of coupling between capsules. Since coupling in general is not desirable, this metric is used to measure the amount of coupling between capsules. A capsule (class or module) should know only about a limited number of other capsules and their services. Otherwise, a tangled web of interdependencies would exist and maintenance would be a nightmare.

OCDO_POLY is a variant of OCDO that treats in a special manner the case of a polymorphic call (i.e. "virtual" C++ methods that are overridden in one or more classes). For this case OCDO_POLY will count both the base class, where the called methods was defined, as well as all the derived classes where the method has been redefined.
