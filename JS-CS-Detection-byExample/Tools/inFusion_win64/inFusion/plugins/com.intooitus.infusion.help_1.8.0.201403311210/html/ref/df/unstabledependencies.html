Unstable Dependencies

Unstable Dependencies are violations of Robert Martin's Stable Dependencies Principle (SDP), applied to subsystems. The SDP affirms that "the dependencies between subsystems in a design should be in the direction of the stability of the subsystems. A subsystem should only depend upon subsystems that are more or at least as stable as it is". Stability is defined in terms of number of reasons to change and number of reasons not to change for a given subsystem. A subsystem that does not depend on many other subsystems but is depended upon by other subsystems, has few reasons to change and respectively many reasons not to change.

Detection rule

The detection rule uses the metric IF (Instability Factor). A subsystem's IF should be larger than the IF of any other subsystem that it depends upon. If this is not the case, we have an instance of Unstable Dependencies.
