Number of Public Methods (NOPUBM)

This metric is defined as the number of public instance methods of a class. The number of public instance methods in a class is a good measure of the amount of responsibility in the class. The number relates tot the amount of collaboration being used. Larger classes may be trying to do too much of the work themselves instead of putting the responsibilities where they belong. They are more complex and harder to maintain. Smaller classes tend to be more reusable, since they provide one set of cohesive services instead of a mixed set of capabilities. All public methods of a class should be implementations of some explicit interface. If this is not the case, exposing public methods of a class that are not part of an interface glues client code to an implementation.
