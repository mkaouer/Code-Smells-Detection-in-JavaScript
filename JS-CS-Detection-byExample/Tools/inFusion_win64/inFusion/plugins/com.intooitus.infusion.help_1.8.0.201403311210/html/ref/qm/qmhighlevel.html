Quality Characteristics

This layer of the QM contains a set of five "quality characteristics" as shown below. The first four are universal and apply to any software system in general, and the last one is restricted to object oriented designs only.



The most significant inspiration sources have been the seminal works of Robert C. Martin [Martin07], Meilir Page-Jones [Page-Jones88] (for procedural programming), Arthur Riel [Riel96], as well as the book on refactoring of Martin Fowler [Fowler99] and the well-know book entitled "Pragmatic Programmer" [Hunt00]. We analyzed in detail all the mentioned literature sources and we noticed that the following design principles: SRP, OCP, LSP, DIP, ISP, REP, CRP, CCP, ADP, SDP, SAP (summarized in [Martin07]) as well as the DRY Principle and the Law of Demeter [Hunt00] are actually expressing exactly the five design properties defined in the QM.
