Data Module

This is the equivalent of the design flaw Data Class in the procedural world. Like a class, a module defines a public interface and has private data and functions. A module that exposes too much of its data and has a low complexity is probably too open for outsiders.

Detection rule

The detection rule employs two metrics, one for measuring the quantity of globally accessible data defined in the module, and a metric for measuring the complexity of the module's functions.

Metrics used

NOGV, WOC
