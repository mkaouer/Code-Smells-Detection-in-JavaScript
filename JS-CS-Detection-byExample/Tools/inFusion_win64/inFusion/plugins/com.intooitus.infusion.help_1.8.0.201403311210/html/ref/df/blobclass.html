Blob Class

Blob Classes are very large and complex, which makes them harder to maintain. Because of their size, they are also more likely to be strongly coupled to other classes in the system and to be non-cohesive. Some of the literature regarding Blob Classes puts a higher emphasis on the "schizophrenia" aspect (the fact that the class implements several responsibilities and is not cohesive), but this aspect is also captured by the Schizophrenic Class design flaw.

Detection rule

Blob Classes are detected using a combination of size, complexity and cohesion metrics, as well as the detection rule for Blob Operation.





Metrics used

CYCLO (indirectly), LOC, LOCOMM (indirectly), MAXNESTING (indirectly), NOAV (indirectly), NOPAR, TCC, WOC
