Cyclomatic Number (CYCLO)

This metric was defined by McCabe as the number of linearly-independent paths through an operation. The cyclomatic number measures the amount of decision logic in a single software module. It is used for two related purposes. First, in the structured testing methodology, it gives the number of recommended tests for software. Second, it is used during all phases of the software lifecycle, beginning with design, to keep software reliable, testable, and manageable.

  Known thresholds
If the maximum value of the extended cyclomatic number is greater than 49, then there are functions, methods and procedures, which are untestable and difficult to understand and are therefore not maintainable.
 A software fragment with a CYCLO value of 1 - 4 has low complexity. A CYCLO value of 5 - 7 represents moderate complexity, which is still easy to understand. A value of 6 - 10 has a high complexity, while a value greater than 10 is very complex and hard to understand.
