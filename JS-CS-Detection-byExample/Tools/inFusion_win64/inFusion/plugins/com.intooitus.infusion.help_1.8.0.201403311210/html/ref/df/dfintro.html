Design Flaws

A "design flaw" denotes a named constellation of structural properties of a fragment of source code, which contradicts one or more of the design principles, rules and best practices, established in software engineering practice. There is a large body of literature describing design flaws, though some authors call them "anti-patterns" or "code smells". Regardless of how they are called, what they all have in common is the fact that they make certain kinds of changes to the code prohibitively expensive, thus affecting in the long run the maintainability of the entire software system.

Detecting design flaws is a tricky proposition, because sometimes a code fragment might be considered flawed in a given context, while in another context, it is justifiable and may not be considered a design flaw at all. In fact, it ultimately depends on the kinds of changes to the code, which can occur during software maintenance and evolution. The detection of design flaws is based on rules, usually combining several software metrics.

List of design flaws
