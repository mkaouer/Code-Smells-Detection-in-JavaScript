74     <h3>Features of CSizingControlBar 2.31</h3>
75     
76     <ul>
77     <li>Resizable control bar, that can be resized both when docked and when floating.
78     <li>Multiple sizing control bars can be docked on the same row/column.
79     <li>Full dynamic resizing, both when docked and floating, including diagonal resizing when floating.
80     <li>State persistence support (SaveState/LoadState).
81     <li>Gripper with "hide bar" flat button.
82     <li>Memory DC flickerless NC painting.
83     <li>Sample extension class with focus autosensing text caption. On Win98/Win2k, the caption is painted with gradient.
84     <li>No custom resources were used (bitmaps, cursors, strings, etc.), so the integration is easier and you have full control over the resources you eventually use in derived classes.
85     <li>Easy to use: just derive your own control bar(s) from CSizingControlBar then add your child controls.
86     </ul>
87     
88     <h3>Instructions</h3>
89     <p>Derive a class from CSizingControlBar (you have an example in mybar.h and mybar.cpp).<br>
90     Add a member variable to CMainFrame (in mainfrm.h).
91     
92     <pre>
93     CMyBar m_wndMyBar;
94     </pre>
95     
96     <p>Create the bar in CMainFrame::OnCreate(). Then set bar styles, enable it to dock... like any control bar. 
97     <p>Some experience in working with control bars (like toolbars) is required.
98     
99     <pre>
100     if (!m_wndMyBar.Create(_T("My Bar"), this, CSize(200, 100),
101         TRUE /*bHasGripper*/, AFX_IDW_CONTROLBAR_FIRST + 32))
102     {
103         TRACE0("Failed to create mybar\n");
104         return -1;
105         // fail to create
106     }
107     m_wndMyBar.SetBarStyle(m_wndMyBar.GetBarStyle() |
108         CBRS_TOOLTIPS | CBRS_FLYBY | CBRS_SIZE_DYNAMIC);
109     m_wndMyBar.EnableDocking(CBRS_ALIGN_ANY);
110     EnableDocking(CBRS_ALIGN_ANY);
111     
112     m_pFloatingFrameClass = RUNTIME_CLASS(CSCBMiniDockFrameWnd);
113     
114     DockControlBar(&m_wndMyBar, AFX_IDW_DOCKBAR_LEFT);
115     </pre>
116     
117     Note: Starting with version 2.3, the line
118     
119     <pre>
120     m_pFloatingFrameClass = RUNTIME_CLASS(CSCBMiniDockFrameWnd);
121     </pre>
122     
123     must be added to CMainFrame::OnCreate(), <i>after</i> the EnableDocking() call. This is required!
124     
125     <h3>Plugging in the CSizingControlBarCF class</h3>
126     
127     <p>1. Add scbarcf.h and scbarcf.cpp to your project.
128     <p>2. In your derived class header file change the "#include "sizecbar.h" to "#include "scbarcf.h".
129     <p>3. Replace the base class for your custom bar from CSizingControlBar to CSizingControlBarCF. If you used the baseCMyBar trick, you only have to change a line. Otherwise, replace all occurences of CSizingControlBar with CSizingControlBarCF in your derived class.
130     <p>4. Rebuild and run. Easy, huh?
131     
132     <h3>Remarks</h3>
133     <p>This class is intended to be used as a base class.
134     Do not simply add your code to the sizecbar.* files - instead create a new class derived from CSizingControlBar and put there what you need.
135     
136     <p><b>Window IDs</b>: You can see above that the control bar is created with the ID AFX_IDW_CONTROLBAR_FIRST + 32.
137     The usage of IDs in the range of AFX_IDW_CONTROLBAR_FIRST + 32 .. AFX_IDW_CONTROLBAR_LAST is required only if the bar will not be enabled for docking (that's is - it will stay fixed right under the frame's menu).
138     But in this situation you won't be able to fully use the features of this class, so if you will enable it to dock (a reasonable guess :) then you can use any valid window ID.<br>
139     Another place where the IDs are important is the saving/loading of the bar's state. 
140     You must use different IDs for each control bar that is enabled to dock, and this includes the other bars too.
141     For example, if you have two toolbars, you can create the first one with the default ID (which is AFX_IDW_TOOLBAR = AFX_IDW_CONTROLBAR_FIRST), but the second one must have a different ID.
142     
143     <p><b>OnUpdateCmdUI</b>: This member function is pure virtual in CControlBar (the base class of CSizingControlBar).
144     Its purpose is to allow updating of controls at idle time (from here CCmdUI::DoUpdate() is called for the toolbars' buttons, dialogbars' controls, the panes of status bar, etc.).<br>
145     However, I found it very useful to update the look of the "x" flat button (no timers needed).
146     So, if you will use this function, don't forget to call the base class' member (see mybar.cpp).
147     
148     <p><b>Dynamic resizing</b>: This feature allows redrawing of the bar during resizing.
149     Also all the bars are repositioned and redrawn if necessary.<br>
150     The SPI_GETDRAGFULLWINDOWS system parameter is queried for this (it is enabled by the "Show window contents while dragging" checkbox in Display Properties).
151     
152     <p><b>CBRS_SIZE_DYNAMIC</b>: This bar style is required. Make sure you add it to the bar, otherwise the application will crash when the user floats a bar. You can add it using SetBarStyle() after Create(), or by changing the default style for Create() to something like: WS_VISIBLE | WS_CHILD | CBRS_TOP | CBRS_SIZE_DYNAMIC.
153     
154     <p><b>State persistence</b>: The common MFC control bars' docking state is saved using CMainFrame::SaveBarState(). In addition to the info saved by this function, the CSizingControlBar class needs to save 3 sizes. This is done in CSizingControlBar::SaveState() function, so a m_wndMyBar.SaveState() call is required. Please note that the state storing code must be placed in CMainFrame's OnClose() or DestroyWindow(), not in OnDestroy(), because at the time WM_DESTROY message is received, the floating bars are already destroyed.<br>
155     In CMainFrame::OnCreate(), the m_wndMyBar.LoadState() call must be placed before LoadBarState().<br>
156     Alternatively, if you have multiple CSizingControlBar derived bars, you can call once the static member SizingControlBar::GlobalSaveState() instead of calling each bar's SaveState(). The same for LoadState() - there is a CSizingControlBar::GlobalLoadState() function. See both samples here for more details.
