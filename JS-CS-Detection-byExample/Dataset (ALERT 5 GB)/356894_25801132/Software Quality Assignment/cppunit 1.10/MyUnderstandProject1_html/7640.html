50     <h3><a name="p1">1 | Preface, Definition</a></h3>
51     
52     <blockquote>
53         <p><u>Definiton:</u><br>
54         A <em>dynamic</em> window is a window that automatically
55         repositions its child controls when its size changes.<br>
56         The opposite behaviour is called <em>static.</em></p>
57         <p>The classes described here will help you implementing <em>dynamic</em>&nbsp;dialogs,
58         formviews, property sheets and control bars.</p>
59         <p>It replaces my former class tree &quot;cdxCSizingDialog, ...&quot;.</p>
60     </blockquote>

94     <h3><a name="p3">3 | Techniques</a></h3>
95     
96     <blockquote>
97       <p>The cdxCDynamicWnd class is a base class of all the ready-to-use classes as
98       cdxCDynamicDialog.<br>
99       To implement the resizing code, you need to tell the cdxCDynamicWnd object,
100       which of your child controls should react on a resizing of your windows and
101       how.</p>
102     
103       <p>To do so, there are now two techniques:</p>
104     
105       <ul>
106         <li><u>Using AddSzControl():<br>
107           </u>As you may know from my former cdxCSizing... classes, you use one of
108           the AddSzControl() overloads to make the dynamic window known to a child
109           window of it.<br>
110           In the upper example window, you would add the following lines to your
111           OnInitDialog() code:</li>
112       </ul>
113       <blockquote>
114         <blockquote>
115           <blockquote>
116             <p><code><font color="#008000">AddSzControl(IDC_BOX_1,mdResize,mdResize);<br>
117             AddSzControl(IDC_LIST,mdResize,mdResize);<br>
118             AddSzControl(IDC_EDIT,mdResize,mdRepos);<br>
119             AddSzControl(IDC_NEW,mdRepos,mdRepos);<br>
120             AddSzControl(IDOK,mdRelative,mdRepos);<br>
121             AddSzControl(IDCANCEL,mdRelative,mdRepos);</font></code></p>
122           </blockquote>
123         </blockquote>
124         <ul>
125           <li>The first argument is the ID of the child control to make known to the
126             dynamic window (note that we do not assign IDC_CHECKBOX since this
127             control does not need to react on changes to the window's size).</li>
128           </ul><ul>      <li>The second argument defines how the control should be treated if the
129             window's width changes,<br>
130             The third defines how to deal with height-changes:<br>
131             You can choose among the following constants:<br>
132             - mdNone (do nothing)<br>
133             - mdRepos (move to left)<br>
134             - mdResize (resize)<br>
135             - mdRelative (keep relative position; e.g. keep centered if control was
136             centered before)</li>
137         </ul>
138         <p>Among others, the following overloads of AddSzControl() are defined (the <i>wnd</i>
139         parameter might be either an ID or a HWND object while a CWnd casts properly
140         to a HWND):</p>
141         <blockquote>
142           <blockquote>
143             <p><code><font color="#008000">AddSzControl(wnd, Mode mdX, Mode mdY);<br>
144             AddSzXControl(wnd, Mode md);<br>
145             AddSzYControl(hwnd, Mode md);</font></code></p>
146           </blockquote>
147         </blockquote>
148         <p>If you are not satisfied with my predefined modes, you can make your own:<br>
149         The following overload takes two bytes for each direction: They defined how
150         much percent of the change in width should be added to the left side of the
151         the child control (<i>x1</i>) and to the right side (<i>x2</i>) (equally for
152         height changes):</p>
153         <blockquote>
154           <blockquote>
155             <p><font color="#008000"><code>AddSzControl(wnd, SBYTE x1, SBYTE y1, SBYTE x2, SBYTE y2);</code><br>
156             <code>AddSzXControl(wnd, SBYTE x1, SBYTE x2);<br>
157             AddSzYControl(wnd, SBYTE y1, SBYTE y2);
158             </code></font></p>
159             <p>My predefined modes have the following values:<br>
160             - mdNone is (<i>x1=</i>0,<i>x2=</i>0)<br>
161             - mdRepos is (100,100)<br>
162             - mdResize is (0,100)<br>
163             - mdRelative is (50,50)</p>
164           </blockquote>
165         </blockquote>
166         <p>Depending on the base class you use the following functions are suitable
167         places to call AddSzControl():</p>
168         <ul>
169           <li>Dialog: OnInitDialog()</li>
170           <li>FormView: OnInitialUpdate()</li>
171           <li>PropPage: OnInitDialog()</li>
172         </ul>
173       </blockquote>
174       <ul>
175         <li><u>Using my new <i>dynamic maps</i>:<br>
176           </u>Some people noted that it is annoying to use the AddSzControl() method
177           since they don't need OnInitDialog() for example - only for the
178           AddSzControl() code.<br>
179           Therefore I added the <i>dynamic maps</i>&nbsp;feature.<br>
180           To use it, add the line DECLARE_DYNAMIC_MAP() to your class definition,
181           and add something like:</li>
182       </ul>
183       <blockquote>
184         <blockquote>
185           <p><code><font color="#008000">BEGIN_DYNAMIC_MAP(CTestDlg,cdxCDynamicDialog)<br>
186           &nbsp;&nbsp;&nbsp; DYNAMIC_MAP_ENTRY(IDC_BOX_1, mdResize, mdResize)<br>
187           &nbsp;&nbsp;&nbsp; DYNAMIC_MAP_ENTRY(IDC_LIST1, mdResize, mdResize)<br>
188           &nbsp;&nbsp;&nbsp; DYNAMIC_MAP_ENTRY(IDC_EDIT, mdResize, mdRepos)<br>
189           &nbsp;&nbsp;&nbsp; DYNAMIC_MAP_ENTRY(IDC_NEW, mdRepos, mdRepos)<br>
190           &nbsp;&nbsp;&nbsp; DYNAMIC_MAP_ENTRY(IDOK, mdRelative, mdRepos)<br>
191           &nbsp;&nbsp;&nbsp; DYNAMIC_MAP_ENTRY(IDCANCEL, mdRelative, mdRepos)<br>
192           END_DYNAMIC_MAP()</font></code></p>
193         </blockquote>
194         <p>If you compare these lines to the above AddSzControl() statements, you'll
195         note that they act similarily.<br>
196         The following macros are defined:</p>
197         <blockquote>
198           <p><code><font color="#008000">DYNAMIC_MAP_ENTRY(ID,MODEX,MODEY)<br>
199           DYNAMIC_MAP_XENTRY(ID,MODEX)<br>
200           DYNAMIC_MAP_YENTRY(ID,MODEY)</font></code></p>
201           <p><code><font color="#008000">DYNAMIC_MAP_ENTRY_EX(ID,X1,Y1,X2,Y2)<br>
202     	DYNAMIC_MAP_XENTRY_EX(ID,X1,X2)<br>
203           DYNAMIC_MAP_YENTRY_EX(ID,Y1,Y2)</font></code></p>
204         </blockquote>
205       </blockquote>
206     </blockquote>

208     <h3><a name="p4">4 | Example: How to create a <em>dynamic</em> dialog</a></h3>
209     
210     <blockquote>
211         <ol>
212             <li><u>A dialog:</u><br>
213                 I suggest that you used your dialog resource editor
214                 to design a dialog as shown in picture #1.<br>
215                 The names of the controls should indicate their
216                 control IDs.<br>
217               <i><u>Important note #1:</u>
218                 Every control that should dynamically move need a
219                 unique ID (=&gt; it is not possible to move static
220                 texts with IDC_STATIC).</i><strong>
221                 <br>
222                 </strong>Moreoever, I assume that you have created a
223                 dialog class called &quot;CTestDlg&quot; for this
224                 dialog.<br>
225               <br>
226                 </li>
227             <li><u>Resizable border and WS_CLIPCHILDREN for your dialog:<br>
228                 </u>Please open your dialog's properties (in the
229                 resource editor), go to the tab &quot;Styles&quot;
230                 and change the &quot;Border&quot; into &quot;resizing&quot;
231                 (otherwise use won't be able to resize your dialog
232                 although it might be dynamic by code).<br>
233               Then, switch to the first tab and activate &quot;clip children&quot;.<br>
234               <br>
235               <b>NOTE</b>: If &quot;clip children&quot; is on and you use <u>group
236               boxes</u>, these boxes need the WS_EX_TRANSPARENT style (can be found
237               at the &quot;extended styles&quot; tab of the group box).<br>
238               This is not a problem with these classes but with the MFC at all !<br>
239               <br>
240                 </li>
241             <li><u>Changing the base-class of your dialog:</u><br>
242                 Open your dialog classes header file, add an<br>
243                 #include &quot;cdxCDynamicDialog.h&quot;<br>
244                 to its head and replace <i> all</i> &quot;CDialog&quot; by
245                 &quot;cdxCDynamicDialog&quot; in both your header and
246                 implementation file.<br>
247               <br>
248                 </li>
249             <li><u>Define how childs should be moved:<br>
250                 </u>Since this release, you can choose among the two techniques
251               described above to implement this behaviour (they can
252                 be mixed if you need that).<br>
253               <br>
254               See <a href="#p3">above</a> to learn how to do so.<br>
255               <br>
256             </li>
257             <li><u>Compile and run.<br>
258               <br>
259               </u>
260             </li>
261             <li>Open your constructor and try to add&nbsp;<code>
262               <font color="#008000">
263               ModifyFlags(flSWPCopyBits,0)</font>&nbsp;</code> to your code.<br>
264               This is an anti-flickering option which does not work with all
265               childconstrols, unfortunately - thus you may need to disable it one
266               time.<br>
267             </li>
268         </ol>
269         <p>Once your code runs, you can modify the behaviour of your class in many
270         ways.</p>
271         <p>The options you may want to make use of are:</p>
272         <ul>
273           <li>Disabling the size icon.</li>
274           <li>Setting a window's minimum and maximum size.</li>
275           <li>Advanced anti-flickering.</li>
276           <li>Advanced AddSzControl() code.</li>
277           <li>Virtual functions that you may like to overload for a more
278             sofisticated <i>dynamic</i> reaction.</li>
279           <li>Default resizing values and many more...</li>
280         </ul>
281         <p>
282         Check the flags, available to you, check functions and if you don't make it
283         work, <a href="mailto:hans.buehler@topmail.de">drop a note to me</a>.</p>
284     </blockquote>
