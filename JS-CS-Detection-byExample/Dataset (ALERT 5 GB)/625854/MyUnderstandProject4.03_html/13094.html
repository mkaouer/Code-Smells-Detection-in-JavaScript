24     /**
25      * FrontEnd is a wrapper class for the chain of front end processors. It provides methods for manipulating and
26      * navigating the processors.
27      * <p/>
28      * The front end is modeled as a series of data processors, each of which performs a specific signal processing
29      * function. For example, a processor performs Fast-Fourier Transform (FFT) on input data, another processor performs
30      * high-pass filtering. Figure 1 below describes how the front end looks like:
31      * <p/>
32      * <center> <img src="doc-files/frontend.jpg"> <br><b>Figure 1: The Sphinx4 front end.</b> </center>
33      * <p/>
34      * Each such data processor implements the {@link edu.cmu.sphinx.frontend.DataProcessor} interface. Objects that
35      * implements the {@link edu.cmu.sphinx.frontend.Data} interface enters and exits the front end, and go between the
36      * processors in the front end. The input data to the front end is typically audio data, but this front end allows any
37      * input type. Similarly, the output data is typically features, but this front end allows any output type. You can
38      * configure the front end to accept any input type and return any output type. We will describe the configuration of
39      * the front end in more detail below.
40      * <p/>
41      * <b>The Pull Model of the Front End</b>
42      * <p/>
43      * The front end uses a pull model. To obtain output from the front end, one would call the method:
44      * <p/>
45      * <code> FrontEnd frontend = ... // see how to obtain the front end below <br>Data output = frontend.getData();
46      * </code>
47      * <p/>
48      * Calling {@link #getData() getData} on the front end would in turn call the getData() method on the last
49      * DataProcessor, which in turn calls the getData() method on the second last DataProcessor, and so on, until the
50      * getData() method on the first DataProcessor is called, which reads Data objects from the input. The input to the
51      * front end is actually another DataProcessor, and is usually (though not necessarily) part of the front end and is not
52      * shown in the figure above. If you want to maintain some control of the input DataProcessor, you can create it
53      * separately, and use the {@link #setDataSource(edu.cmu.sphinx.frontend.DataProcessor) setDataSource} method to set it
54      * as the input DataProcessor. In that case, the input DataProcessor will be prepended to the existing chain of
55      * DataProcessors. One common input DataProcessor is the {@link edu.cmu.sphinx.frontend.util.Microphone}, which
56      * implements the DataProcessor interface.
57      * <p/>
58      * <code> DataProcessor microphone = new Microphone(); <br>microphone.initialize(...);
59      * <br>frontend.setDataSource(microphone); </code>
60      * <p/>
61      * Another common input DataProcessor is the {@link edu.cmu.sphinx.frontend.util.StreamDataSource}. It turns a Java
62      * {@link java.io.InputStream} into Data objects. It is usually used in batch mode decoding.
63      * <p/>
64      * <b>Configuring the front end</b>
65      * <p/>
66      * The front end must be configured through the Sphinx properties file. For details about configuring the front end,
67      * refer to the document <a href="doc-files/FrontEndConfiguration.html">Configuring the Front End</a>.
68      * <p/>
69      * Current state-of-the-art front ends generate features that contain Mel-frequency cepstral coefficients (MFCC). To
70      * specify such a front end (called a 'pipeline') in Sphinx-4, insert the following lines in the Sphinx-4 configuration
71      * file:
72      * <p/>
73      * <pre>
74      * &lt;component name="mfcFrontEnd" type="edu.cmu.sphinx.frontend.FrontEnd"&gt;
75      *     &lt;propertylist name="pipeline"&gt;
76      *        &lt;item&gt;preemphasizer&lt;/item&gt;
77      *        &lt;item&gt;windower&lt;/item&gt;
78      *        &lt;item&gt;dft&lt;/item&gt;
79      *        &lt;item&gt;melFilterBank&lt;/item&gt;
80      *        &lt;item&gt;dct&lt;/item&gt;
81      *        &lt;item&gt;batchCMN&lt;/item&gt;
82      *        &lt;item&gt;featureExtractor&lt;/item&gt;
83      *     &lt;/propertylist&gt;
84      * &lt;/component&gt;
85      * <p/>
86      * &lt;component name="preemphasizer" type="{@link edu.cmu.sphinx.frontend.filter.Preemphasizer
87      * edu.cmu.sphinx.frontend.filter.Preemphasizer}"/&gt;
88      * &lt;component name="windower" type="{@link edu.cmu.sphinx.frontend.window.RaisedCosineWindower
89      * edu.cmu.sphinx.frontend.window.RaisedCosineWindower}"/&gt;
90      * &lt;component name="dft" type="{@link edu.cmu.sphinx.frontend.transform.DiscreteFourierTransform
91      * edu.cmu.sphinx.frontend.transform.DiscreteFourierTransform}"/&gt;
92      * &lt;component name="melFilterBank" type="{@link edu.cmu.sphinx.frontend.frequencywarp.MelFrequencyFilterBank
93      * edu.cmu.sphinx.frontend.frequencywarp.MelFrequencyFilterBank}"/&gt;
94      * &lt;component name="dct" type="{@link edu.cmu.sphinx.frontend.transform.DiscreteCosineTransform
95      * edu.cmu.sphinx.frontend.transform.DiscreteCosineTransform}"/&gt;
96      * &lt;component name="batchCMN" type="{@link edu.cmu.sphinx.frontend.feature.BatchCMN
97      * edu.cmu.sphinx.frontend.feature.BatchCMN}"/&gt;
98      * &lt;component name="featureExtractor" type="{@link edu.cmu.sphinx.frontend.feature.DeltasFeatureExtractor
99      * edu.cmu.sphinx.frontend.feature.DeltasFeatureExtractor}"/&gt;
100      * </pre>
101      * <p/>
102      * Note: In this example, 'mfcFrontEnd' becomes the name of the front end.
103      * <p/>
104      * Sphinx-4 also allows you to: <ul> <li>specify multiple front end pipelines</li> <li>specify multiple instance of the
105      * same DataProcessor in the same pipeline</li> </ul>
106      * <p/>
107      * For details on how to do this, refer to the document <a href="doc-files/FrontEndConfiguration.html">Configuring the
108      * Front End</a>.
109      * <p/>
110      * <b>Obtaining a Front End</b>
111      * <p/>
112      * In order to obtain a front end, it must be specified in the configuration file. The Sphinx-4 front end is connected
113      * to the rest of the system via the scorer. We will continue with the above example to show how the scorer will obtain
114      * the front end. In the configuration file, the scorer should be specified as follows:
115      * <p/>
116      * <pre>
117      * &lt;component name="scorer" type="edu.cmu.sphinx.decoder.scorer.SimpleAcousticScorer"&gt;
118      *     &lt;property name="frontend" value="mfcFrontEnd"/&gt;
119      * &lt;/component&gt;
120      * </pre>
121      * <p/>
122      * In the SimpleAcousticScorer, the front end is obtained in the {@link edu.cmu.sphinx.util.props.Configurable#newProperties
123      * newProperties} method as follows:
124      * <pre>
125      * public void newProperties(PropertySheet ps) throws PropertyException {
126      *     FrontEnd frontend = (FrontEnd) ps.getComponent("frontend", FrontEnd.class);
127      * }
128      * </pre>
129      */
130     public class FrontEnd extends BaseDataProcessor {
131     
132         /** the name of the property list of all the components of the frontend pipe line */
133         @S4ComponentList(type = DataProcessor.class, beTolerant = true)
134         public final static String PROP_PIPELINE = "pipeline";
135     
136     
137         // ----------------------------
138         // Confiugration data
139         // -----------------------------
140         private List<DataProcessor> frontEndList;
141     
142         private DataProcessor first;
143         private DataProcessor last;
144         private Vector<SignalListener> signalListeners = new Vector<SignalListener>();
145     
146     
147         /* (non-Javadoc)
148          * @see edu.cmu.sphinx.util.props.Configurable#newProperties(edu.cmu.sphinx.util.props.PropertySheet)
149          */
150         public void newProperties(PropertySheet ps) throws PropertyException {
151             super.newProperties(ps);
152             frontEndList = (List<DataProcessor>) ps.getComponentList(PROP_PIPELINE);
153     
154             last = null;
155             for (DataProcessor dp : frontEndList) {
156                 assert dp != null;
157     
158                 if (last != null)
159                     dp.setPredecessor(last);
160     
161                 if (first == null) {
162                     first = dp;
163                 }
164                 last = dp;
165             }
166             initialize();
167         }
168     
169     
170         /* (non-Javadoc)
171         * @see edu.cmu.sphinx.frontend.DataProcessor#initialize(edu.cmu.sphinx.frontend.CommonConfig)
172         */
173         public void initialize() {
174             super.initialize();
175             for (Iterator<DataProcessor> i = frontEndList.iterator(); i.hasNext();) {
176                 DataProcessor dp = i.next();
177                 dp.initialize();
178             }
179         }
180     
181     
182         /**
183          * Sets the source of data for this front end. It basically sets the predecessor of the first DataProcessor of this
184          * front end.
185          *
186          * @param dataSource the source of data
187          */
188         public void setDataSource(DataProcessor dataSource) {
189             first.setPredecessor(dataSource);
190         }
191     
192     
193         /** Returns the collection of <code>DataProcessor</code>s which setup this <code>FrontEnd</code>. */
194         public List<DataProcessor> getElements() {
195             return frontEndList;
196         }
197     
198     
199         /**
200          * Returns the processed Data output, basically calls <code>getData()</code> on the last processor.
201          *
202          * @return an Data object that has been processed by this front end
203          * @throws DataProcessingException if a data processor error occurs
204          */
205         public Data getData() throws DataProcessingException {
206             Data data = last.getData();
207     
208             // fire the signal listeners if its a signal
209             if (data instanceof Signal) {
210                 fireSignalListeners((Signal) data);
211             }
212     
213             return data;
214         }
215     
216     
217         /**
218          * Sets the source of data for this front end. It basically calls <code>setDataSource(dataSource)</code>.
219          *
220          * @param dataSource the source of data
221          */
222         public void setPredecessor(DataProcessor dataSource) {
223             setDataSource(dataSource);
224         }
225     
226     
227         /**
228          * Add a listener to be called when a signal is detected.
229          *
230          * @param listener the listener to be added
231          */
232         public void addSignalListener(SignalListener listener) {
233             signalListeners.add(listener);
234         }
235     
236     
237         /**
238          * Removes a listener for signals.
239          *
240          * @param listener the listener to be removed
241          */
242         public void removeSignalListener(SignalListener listener) {
243             signalListeners.remove(listener);
244         }
245     
246     
247         /**
248          * Fire all listeners for signals.
249          *
250          * @param signal the signal that occurred
251          */
252         protected void fireSignalListeners(Signal signal) {
253             Vector<SignalListener> copy = new Vector<SignalListener>(signalListeners);
254             for (SignalListener listener : copy) {
255                 listener.signalOccurred(signal);
256             }
257         }
258     
259     
260         /** Returns the last data procssor within the <code>DataProcessor</code> chain of this <code>FrontEnd</code>. */
261         public DataProcessor getLastDataProcessor() {
262             return last;
263         }
264     
265     
266         /**
267          * Returns a description of this FrontEnd in the format: <front end name> {<DataProcessor1>, <DataProcessor2> ...
268          * <DataProcessorN>}
269          *
270          * @return a description of this FrontEnd
271          */
272         public String toString() {
273             String description = "";
274             DataProcessor current = last;
275             while (current != null) {
276                 description = (current + description);
277                 current = current.getPredecessor();
278                 if (current != null) {
279                     description = (", " + description);
280                 }
281             }
282             return (super.toString() + " {" + description + "}");
283         }
284     
285     }
