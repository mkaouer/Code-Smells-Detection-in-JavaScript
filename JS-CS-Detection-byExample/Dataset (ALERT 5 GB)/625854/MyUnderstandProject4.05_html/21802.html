35     /**
36      * A linguist that can represent large vocabularies efficiently. This class implements the Linguist interface. The main
37      * role of any linguist is to represent the search space for the decoder. The initial state in the search space can be
38      * retrieved by a SearchManager via a call to <code> getInitialSearchState</code>. This method returns a SearchState.
39      * Successor states can be retrieved via calls to <code>SearchState.getSuccessors().</code>. There are a number of
40      * search state sub-interfaces that are used to indicate different types of states in the search space:
41      * <p/>
42      * <ul> <li><b>WordSearchState </b>- represents a word in the search space. <li><b>UnitSearchState </b>- represents a
43      * unit in the search space <li><b>HMMSearchState </b> represents an HMM state in the search space </ul>
44      * <p/>
45      * A linguist has a great deal of latitude about the order in which it returns states. For instance a 'flat' linguist
46      * may return a WordState at the beginning of a word, while a 'tree' linguist may return WordStates at the ending of a
47      * word. Likewise, a linguist may omit certain state types completely (such as a unit state). Some Search Managers may
48      * want to know a priori the order in which states will be generated by the linguist. The method
49      * <code>getSearchStateOrder</code> can be used to retrieve the order of state returned by the linguist.
50      * <p/>
51      * <p/>
52      * Depending on the vocabulary size and topology, the search space represented by the linguist may include a very large
53      * number of states. Some linguists will generate the search states dynamically, that is, the object representing a
54      * particular state in the search space is not created until it is needed by the SearchManager. SearchManagers often
55      * need to be able to determine if a particular state has been entered before by comparing states. Because SearchStates
56      * may be generated dynamically, the <code>SearchState.equals()</code> call (as opposed to the reference equals '=='
57      * method) should be used to determine if states are equal. The states returned by the linguist will generally provide
58      * very efficient implementations of <code>equals</code> and <code>hashCode</code>. This will allow a SearchManager to
59      * maintain collections of states in HashMaps efficiently.
60      * <p/>
61      * <p/>
62      * <p/>
63      * <b>LexTeeLinguist Characteristics </b>
64      * <p/>
65      * Some characteristics of this linguist: <ul> <li><b>Dynamic </b>- the linguist generates search states on the fly,
66      * greatly reducing the required memory footprint <li><b>tree topology </b> this linguist represents the search space as
67      * an inverted tree. Units near the roots of word are shared among many different words. These reduces the amount of
68      * states that need to be considered during the search. <li><b>HMM sharing </b>- because of state tying in the acoustic
69      * models, it is often the case that triphone units that differ in the right context actually are represented by the
70      * same HMM. This linguist recognizes this case and will use a single state to represent the HMM instead of two states.
71      * This can greatly reduce the number of states generated by the linguist. <li><b>Small-footprint </b>- this linguist
72      * uses a few other techniques to reduce the overall footprint of the search space. One technique that is particularly
73      * helpful is to share the end word units (where the largest fanout of states occurs) across all of the words. For a 60K
74      * word vocabulary, these can result in a reduction in tree nodes of about 2 million to around 3,000. <li><b>Quick
75      * loading </b>- this linguist can compile the search space very quickly. A 60K word vocabulary can be made ready in
76      * less than 10 seconds. </ul>
77      * <p/>
78      * This linguist is not a general purpose linguist. It does impose some constraints:
79      * <p/>
80      * <ul> <li><b>unit size </b>- this linguist will units that are no larger than triphones. <li><b>n-gram grammars </b>-
81      * this linguist will generate the search space directly from the N-Gram language model. The vocabulary supported is the
82      * intersection of the words found in the language model and the words that exist in the Dictionary. It is assumed that
83      * all sequences of words in the vocabulary are valid. This linguist doesn't support arbitrary grammars. </ul>
84      * <p/>
85      * <p/>
86      * <b>Design Notes </b> The following are some notes describing the design of this linguist. They may be helpful to
87      * those who want to understand how this linguist works but are not necessary if you are only interested in using this
88      * linguist.
89      * <p/>
90      * <p/>
91      * <p/>
92      * <b>Search Space Representation </b> It has been shown that representing the search space as a tree can greatly reduce
93      * the number of active states in a search since the units at the beginnings of words can be shared across multiple
94      * words. For example, with a large vocabulary (60K words), at the end of a word, with a flat representation, we have to
95      * provide transitions to the initial state of each possible word. That is 60K transitions. In a tree based system we
96      * need to only provide transitions to each initial phone (within its context). That is about 1600 transitions. This is
97      * a substantial reduction. Conceptually, this tree consists of a node for each possible initial unit. Each node can
98      * have an arbitrary number of children which can be either unit nodes or word nodes.
99      * <p/>
100      * <p/>
101      * This linguist uses the HMMTree class to build and represent the tree. The HMMTree is given the dictionary and
102      * language model and builds the lextree. Instead of representing the nodes in the tree as phonemes and words as is
103      * typically done, the HMMTree represents the tree as HMMs and words. The HMM is essentially a unit within its context.
104      * This is typically a triphone (although for some units (such as SIL) it is a simple phone. Representing the nodes as
105      * HMM instead of nodes yields a much larger tree, but also has some advantages:
106      * <p/>
107      * <ul> <li>Because of state-tying in the acoustic models, many distinct triphones actually share an HMM. Representing
108      * the nodes as HMMs allows these shared HMMs to be represented in the tree only once instead of many times if we
109      * representing states as phones or triphones. This leads to a reduction in the actual number of states that are
110      * considered during a search. Experiments have shown that this can reduce the required beam by a factor of 2 or 3.
111      * <li>By representing the nodes as HMM, we avoid having to lookup the HMM for a particular triphone during the search.
112      * This is a modest savings. </ul>
113      * <p/>
114      * There are some disadvantages in representing the tree with HMMs:
115      * <p/>
116      * <ul> <li><b>size</b> since HMMs represent units in their context, we have many more copies of each node. For
117      * instance, instead of having a single unit representing the initial 'd' in the word 'dog' we would have about 40 HMMs,
118      * one for each possible left context. <li><b>speed </b> building the much larger HMM tree can take much more time,
119      * since many more nodes are needed to represent the tree. <li><b>complexity </b> representing the tree with HMMs is
120      * more complex. There are multiple entry points for each word/unit that have to be dealt with. </ul>
121      * <p/>
122      * Luckily the size and speed issues can be mitigated (by adding a bit more complexity of course). The bulk of the nodes
123      * in the HMM tree are the word ending nodes. There is a word ending node for each possible right context. To reduce
124      * space, all of the word ending nodes are replaced by a single EndNode. During the search, the actual HMM nodes for a
125      * particular EndNode are generated on request. These sets of HMM nodes can be shared among different word endings, and
126      * therefore are cached. The effect of using this EndNode optimization is to reduce the space required by the tree by
127      * about 300mb and the time required to generate the tree from about 60 seconds to about 6 seconds.
128      *
129      * <p/>
130      * <b>Word Histories </b>
131      * <p/>
132      * We use explicit backoff for word histories. That technique is proven to be useful and save number of
133      * states. The reasoning is the following. With a vocabulary of size N, you have N^2 unique bigram 
134      * histories. So the token stack will have N^2*K unique tokens, where K is the number of states per token.
135      * For a 100k vocab, 3 states per HMM, that will be 3*10^10 tokens (max). Of course, a large majority 
136      * of them will be pruned, but really, its still way too much. If you stick with the <b>actual</b>  K-gram 
137      * used (i.e. accounting explicitly for backoff), then this reduces <b>tremendously</b>.
138      * Most bigrams dont have corresponding trigrams.  Not all 10^10 bigrams have trigrams. We only 
139      * need to store as many explicit tokens as the number of bigrams that have trigrams.
140      */
141     public class LexTreeLinguist implements Linguist {

402          * retrieves the initial language state
403          *
404          * @return the initial language state

406         private SearchState getInitialSearchState() {
407             InitialWordNode node = hmmTree.getInitialNode();
408             assert node != null : "Initial state <s> is missing in the language model";
409             return new LexTreeWordState(node, node.getParent(), (new WordSequence(sentenceStartWordArray)).trim(
410                     maxDepth - 1), 0f, logOne, logOne, false);
411         }

414         /** Compiles the n-gram into a lex tree that is used during the search */
415         private void compileGrammar() {
416             TimerPool.getTimer(this, "Compile").start();
417     
418             sentenceEndWord = dictionary.getSentenceEndWord();
419             sentenceStartWordArray = new Word[1];
420             sentenceStartWordArray[0] = dictionary.getSentenceStartWord();
421             maxDepth = languageModel.getMaxDepth();
422     
423             generateHmmTree();
424     
425             TimerPool.getTimer(this,"Compile").stop();
426             // Now that we are all done, dump out some interesting
427             // information about the process
428     
429             searchGraph = new LexTreeSearchGraph(getInitialSearchState());
430         }

434             hmmPool = new HMMPool(acousticModel, logger, unitManager);
435     
436             hmmTree = new HMMTree(hmmPool, dictionary, languageModel, addFillerWords, languageWeight);
437     
438             hmmPool.dumpInfo();
439         }

481         /**
482          * The LexTreeLinguist returns language states to the search manager. This class forms the base implementation for
483          * all language states returned. This LexTreeState keeps track of the probability of entering this state (a
484          * language+insertion probability) as well as the unit history. The unit history consists of the LexTree nodes that
485          * correspond to the left, center and right contexts.
486          * <p/>
487          * This is an abstract class, subclasses must implement the getSuccessorss method.
488          */
489         abstract class LexTreeState implements SearchState, SearchStateArc {
490     
491             private final Node node;
492             private final WordSequence wordSequence;
493             protected final boolean collapsed;
494             final float currentSmearTerm;
495             final float currentSmearProb;
496     
497     
498             /**
499              * Creates a LexTreeState.
500              *
501              * @param node         the node associated with this state
502              * @param wordSequence the history of words up until this point
503     
504     
505              */
506             LexTreeState(Node node, WordSequence wordSequence, float smearTerm,
507                          float smearProb, boolean collapsed) {
508                 this.node = node;
509                 this.wordSequence = wordSequence;
510                 currentSmearTerm = smearTerm;
511                 currentSmearProb = smearProb;
512                 this.collapsed = collapsed;
513             }
514     
515     
516             /**
517              * Gets the unique signature for this state. The signature building code is slow and should only be used for
518              * non-time-critical tasks such as plotting states.
519              *
520              * @return the signature
521              */
522             @Override
523             public String getSignature() {
524                 return "lts-" + node.hashCode() + "-ws-" + wordSequence;
525             }
526     
527     
528             public float getSmearTerm() {
529                 return currentSmearTerm;
530             }
531     
532     
533             public float getSmearProb() {
534                 return currentSmearProb;
535             }
536     
537     
538             /**
539              * Generate a hashcode for an object
540              *
541              * @return the hashcode
542              */
543             @Override
544             public int hashCode() {
545                 int hashCode = collapsed ? 37 : wordSequence.hashCode() * 37;
546                 hashCode += node.hashCode();
547                 return hashCode;
548             }
549     
550     
551             /**
552              * Determines if the given object is equal to this object
553              *
554              * @param o the object to test
555              * @return <code>true</code> if the object is equal to this
556              */
557             @Override
558             public boolean equals(Object o) {
559                 if (o == this) {
560                     return true;
561                 } else if (o instanceof LexTreeState) {
562                     LexTreeState other = (LexTreeState) o;
563                     if (node != other.node)
564                         return false;
565                     if (collapsed != other.collapsed) {
566                         return false;
567                     }
568                     boolean wordSequenceMatch = collapsed || wordSequence
569                             .equals(other.wordSequence);
570                     return wordSequenceMatch;
571                 } else {
572                     return false;
573                 }
574             }

693                 Node[] nodes = theNode.getSuccessors();
694                 SearchStateArc[] arcs = new SearchStateArc[nodes.length];
695                 // System.out.println("Arc: "+ this);
696                 int i = 0;
697                 for (Node nextNode : nodes) {
698                     //  System.out.println(" " + nextNode);
699                     if (nextNode instanceof WordNode) {
700                         arcs[i] = createWordStateArc((WordNode) nextNode,
701                                 (HMMNode) getNode(), this);
702                     } else if (nextNode instanceof EndNode) {
703                         arcs[i] = createEndUnitArc((EndNode) nextNode, this);

714              * Creates a word search state for the given word node
715              *
716              * @param wordNode the wordNode
717     
718     
719              * @return the search state for the wordNode
720              */
721             protected SearchStateArc createWordStateArc(WordNode wordNode,
722                                                         HMMNode lastUnit, LexTreeState previous) {
723                 // System.out.println("CWSA " + wordNode + " fup " + fixupProb);
724                 float languageProbability = logOne;
725                 Word nextWord = wordNode.getWord();
726                 float smearTerm = previous.getSmearTerm();
727                 
728                 if (nextWord.isFiller() && nextWord != sentenceEndWord) {
729                     return new LexTreeWordState(wordNode, lastUnit,
730                             wordSequence,
731                             smearTerm, logOne, languageProbability, collapsed);                
732                 }
733       
734                 WordSequence nextWordSequence = wordSequence.addWord(nextWord, maxDepth);
735                 ProbDepth probDepth = languageModel.getProbDepth(nextWordSequence);
736                 smearTerm = getSmearTermFromLanguageModel(nextWordSequence);
737                 // System.out.println("LP " + nextWordSequence + " " +
738                 // logProbability);
739                 float probability = probDepth.probability * languageWeight;
740                 // subtract off the previously applied smear probability
741                 languageProbability = probability - previous.getSmearProb();
742       
743                 boolean collapse = (probDepth.depth < maxDepth - 1) || !fullWordHistories;
744                 
745                 if (nextWord == sentenceEndWord) {
746                     return new LexTreeEndWordState(wordNode, lastUnit,
747                             nextWordSequence.trim(maxDepth - 1),
748                             smearTerm, logOne, languageProbability, collapse);
749                 }
750                 
751                 return new LexTreeWordState(wordNode, lastUnit,
752                             nextWordSequence.trim(maxDepth - 1),
753                             smearTerm, logOne, languageProbability, collapse);
754             }

763              */

765                 SearchStateArc arc;
766                 // System.out.println("CUSA " + hmmNode);
767                 float insertionProbability = calculateInsertionProbability(hmmNode);
768                 float smearProbability = getUnigramSmear(hmmNode)
769                         + previous.getSmearTerm();
770                 float languageProbability = smearProbability - previous.getSmearProb();
771     
772                 // if we want a unit state create it, otherwise
773                 // get the first hmm state of the unit
774     
775                 if (generateUnitStates) {
776                     arc = new LexTreeUnitState(hmmNode, getWordHistory(), previous
777                             .getSmearTerm(), smearProbability, languageProbability,
778                             insertionProbability, collapsed);
779                 } else {
780                     HMM hmm = hmmNode.getHMM();
781                     arc = new LexTreeHMMState(hmmNode, getWordHistory(), previous
782                             .getSmearTerm(), smearProbability, hmm.getInitialState(),
783                             languageProbability, insertionProbability, null, collapsed);
784                 }
785                 return arc;
786             }

1295                             if (arc.getHMMState().isEmitting()) {
1296                                 // if its a self loop and the prob. matches
1297                                 // reuse the state
1298                                 if (arc.getHMMState() == hmmState
1299                                         && logInsertionProbability == arc
1300                                         .getLogProbability()) {
1301                                     nextStates[i] = this;
1302                                 } else {
1303                                     nextStates[i] = new LexTreeHMMState(
1304                                             (HMMNode) getNode(), getWordHistory(),
1305                                             getSmearTerm(), getSmearProb(),
1306                                             arc.getHMMState(), logOne,

1313                                         arc.getHMMState(),
1314                                         arc.getLogProbability(), parentNode, collapsed);

1316                         }
1317                     }

1492                         int index = 0;

1511                         // now add the link to the end of sentence arc:
1512

1573          *
1574          * @param unitNode the unit lex node
1575          * @return the insertion probability
1576          */
1577         private float calculateInsertionProbability(UnitNode unitNode) {
1578             int type = unitNode.getType();
1579     
1580             if (type == UnitNode.SIMPLE_UNIT) {
1581                 return logUnitInsertionProbability;
1582             } else if (type == UnitNode.WORD_BEGINNING_UNIT) {
1583                 return logUnitInsertionProbability + logWordInsertionProbability;
1584             } else if (type == UnitNode.SILENCE_UNIT) {
1585                 return logSilenceInsertionProbability;
1586             } else { // must be filler
1587                 return logFillerInsertionProbability;
1588             }
