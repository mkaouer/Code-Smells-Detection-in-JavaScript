18       /**
19        * A test case defines the fixture to run multiple tests. To define a test case
20        * do the following:
21        * - implement a subclass of TestCase 
22        * - define instance variables that store the state of the fixture
23        * - initialize the fixture state by overriding setUp
24        * - clean-up after a test by overriding tearDown.
25        *
26        * Each test runs in its own fixture so there
27        * can be no side effects among test runs.
28        * Here is an example:
29        * 
30        * \code
31        * class MathTest : public TestCase {
32        *     protected: int m_value1;
33        *     protected: int m_value2;
34        *
35        *     public: MathTest (string name)
36        *                 : TestCase (name) {
37        *     }
38        *
39        *     protected: void setUp () {
40        *         m_value1 = 2;
41        *         m_value2 = 3;
42        *     }
43        * }
44        * \endcode
45        *
46        * For each test implement a method which interacts
47        * with the fixture. Verify the expected results with assertions specified
48        * by calling assert on the expression you want to test:
49        * 
50        * \code
51        *    protected: void testAdd () {
52        *        int result = value1 + value2;
53        *        assert (result == 5);
54        *    }
55        * \endcode
56        * 
57        * Once the methods are defined you can run them. To do this, use
58        * a TestCaller.
59        *
60        * \code
61        * Test *test = new TestCaller<MathTest>("testAdd", MathTest::testAdd);
62        * test->run ();
63        * \endcode
64        *
65        *
66        * The tests to be run can be collected into a TestSuite. CppUnit provides
67        * different test runners which can run a test suite and collect the results.
68        * The test runners expect a static method suite as the entry
69        * point to get a test to run.
70        * 
71        * \code
72        * public: static MathTest::suite () {
73        *      TestSuite *suiteOfTests = new TestSuite;
74        *      suiteOfTests->addTest(new TestCaller<MathTest>(
75        *                              "testAdd", testAdd));
76        *      suiteOfTests->addTest(new TestCaller<MathTest>(
77        *                              "testDivideByZero", testDivideByZero));
78        *      return suiteOfTests;
79        *  }
80        * \endcode
81        * 
82        * Note that the caller of suite assumes lifetime control
83        * for the returned suite.
84        *
85        * \see TestResult
86        * \see TestSuite 
87        * \see TestCaller
88        *
89        */
